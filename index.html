Overview

What this tutorial teaches

Git Add

Stages your modified files so Git knows which changes you want to save.

Git Commit

Saves your changes locally with a comment or message explaining what changed.

Git Pull

Downloads the latest changes from GitHub to your local machine.

Git Push

Uploads your committed changes from your computer to GitHub.

Workflow

Complete Git process for uploading and syncing

01

Check your work

Before anything else, check what files were changed.

git status
02

Add files to staging

This prepares all your updated files for commit.

git add .
03

Commit with a comment

Save your update locally and add a message describing the change.

git commit -m "Update: fixed navigation style"
04

Pull latest changes

Get the newest version from GitHub before pushing your own changes.

git pull origin main
05

Push to GitHub

Upload your committed changes to your remote repository.

git push origin main
06

Update and repeat

Edit files, add changes, commit again, then pull and push when needed.

git add .
git commit -m "Update: added new section"
git pull origin main
git push origin main

Tips

Instructor notes

1. Always check first

Use git status before committing so you know what changed.

2. Write clear comments

Use meaningful commit messages like "Update header design" or "Fix login bug".

3. Pull before pushing

When working with a team, pull the latest changes first to avoid conflicts.

4. Commit often

Smaller commits are easier to track, understand, and troubleshoot.
